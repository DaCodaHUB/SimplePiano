# SimplePiano
**SimplePiano** is a native Android application built entirely in **Kotlin**. It demonstrates a clean, modern approach to mobile development, focusing on responsive UI and real-time user interaction.
> 🎹 **Status:** Active Development / Portfolio Piece
> 
## 🛠 Tech Stack
 * **Language:** Kotlin (100%) - Leveraging modern language features for concise and safe code.
 * **Build System:** Gradle (Kotlin Script - .kts) - Using the latest standard for build configurations.
 * **Target Platform:** Android (Native)
## 🧱 Project Structure
The project follows the standard Android folder hierarchy, optimized for clarity and separation of concerns:
 * **/app**: Contains the core logic, UI components, and resource files.
 * **/gradle**: Standard wrapper and dependency management.
 * **build.gradle.kts**: Centralized dependency and version management using Kotlin DSL.
## 🚀 Key Features (Current & In-Progress)
 * **Native Performance:** Built with 100% Kotlin to ensure the fastest possible response time between touch input and audio output.
 * **Modern Build Configuration:** Uses build.gradle.kts and settings.gradle.kts, reflecting current Android development best practices (2025/2026 standards).
 * **Clean Implementation:** Focused on a minimal codebase that is easy to scale and audit for technical interviews.
## 📖 Why This Project?
This repository serves as a focused study on **Mobile Interaction Design**.
While many apps focus on data fetching, **SimplePiano** addresses the unique challenges of:
 1. **Low-Latency Feedback:** Ensuring that the time between a user touching a key and hearing a note is imperceptible.
 2. **Concurrency:** Handling multiple simultaneous inputs across the device screen.
 3. **Resource Management:** Efficiently handling audio assets within the Android lifecycle.
## 🛠 Installation & Setup
 1. Clone the repository:
   ```bash
   git clone https://github.com/DaCodaHUB/SimplePiano.git
   
   ```
 2. Open the project in **Android Studio (Ladybug or newer)**.
 3. Sync Gradle and run the app on an emulator or physical device.
